Scope & hoisting

How JavaScript handles variable visibility and declarations

Global scope
Accessible everywhere
Variables declared outside any function or block are globally accessible.
let a = 10; function show() { console.log(a); }
✓ Works fine
Block scope
let & const stay inside { }
Variables declared with let or const are only accessible within their block.
{ let x = 5; } console.log(x);
✗ ReferenceError
Hoisting
Declarations move to top
JS moves declarations to the top before running. let/const are hoisted but NOT initialised — accessing them throws. var is hoisted AND initialised as undefined.
console.log(a); let a = 10;
✗ ReferenceError (TDZ)
