Scope & closures

Where variables live, how JS searches for them, and how closures remember values

Global scope
Accessible everywhere
Variables declared outside any function or block are available in the entire file.
let x = 10; function show() { console.log(x); // ✓ works }
→ 10
Function scope
Trapped inside the function
Variables declared inside a function exist only during that function's execution.
function test() { let age = 20; } console.log(age); // ✗ error
→ ReferenceError
Block scope
let & const inside { }
let and const are block-scoped — they don't escape if, for, or any { } block.
if (true) { let x = 10; } console.log(x); // ✗ error
→ ReferenceError
Lexical scope
Inner sees outer
A nested function can read variables from its parent — but the parent cannot reach inside the child.
function outer() { let a = 10; function inner() { console.log(a); // ✓ } inner(); } outer();
→ 10
Scope chain
JS searches upward
When a variable isn't found locally, JS walks up through parent scopes until it hits global — or throws.
local (inner) → z
↑ not found? look up
parent (a) → y
↑ not found? look up
global → x
let x = 1; function a() { let y = 2; function b() { let z = 3; console.log(x, y, z); } b(); } a();
→ 1 2 3
Closures
Function remembers outer vars
The returned function keeps a live reference to count even after counter() finishes — this is a closure.
function counter() { let count = 0; return function() { count++; console.log(count); }; } const c = counter(); c(); // 1 c(); // 2
→ 1 then 2
