Advanced functions

Higher-order functions, callbacks, IIFE, recursion, and function methods

Higher-order functions
Receive or return a function
A function that takes another function as an argument or returns one is called a higher-order function.
function run(fn) { fn(); } run(() => console.log("Hello"));
→ Hello
Callback functions
Function passed as argument
The function you pass in is called a callback — the receiver calls it when ready. Used heavily in arrays and async.
function process(cb) { cb(); } process(function() { console.log("Done"); });
→ Done
IIFE
Runs immediately on define
Immediately Invoked Function Expression — wrap in ( ) then call with (). Variables inside don't leak to global.
// regular function (function() { console.log("Run"); })(); // arrow version (() => { console.log("Run"); })();
→ Run
Recursion
Function calls itself
A function that calls itself until a base condition stops it. Must always have a base case or it runs forever.
function count(n) { if (n === 0) return; // base case console.log(n); count(n - 1); // recursive call } count(3);
→ 3 2 1
Function methods
call / apply / bind
Used to manually control what this refers to when calling a function.
call()call with specific this, args one by one
apply()same as call, but args as array
bind()returns new function with this locked in
const obj = { name: "Sajin" }; function greet() { console.log(this.name); } greet.call(obj);
→ Sajin
