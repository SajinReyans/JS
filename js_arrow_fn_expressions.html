Function expressions & arrow functions

Two alternate ways to write functions — with key differences in hoisting and this

Function expression
Store function in variable
Assign a function to a const/let. Not hoisted — must be defined before calling.
const greet = function() { console.log("Hello"); }; greet();
→ Hello
Not hoisted
Call after only
Unlike declarations, function expressions cannot be called before the line they appear on.
greet(); // ✗ Error const greet = function() { console.log("Hello"); };
→ ReferenceError
Arrow function
Shorter syntax with =>
Arrow functions are compact. Single param → no brackets. Single expression → no return.
// no params const greet = () => console.log("Hi"); // one param — no ( ) needed const square = x => x * x; // multiple params const add = (a, b) => a + b;
Implicit return
Skip the return keyword
If the body is a single expression (no { }), the result is returned automatically.
// explicit (with braces) const add = (a, b) => { return a + b; }; // implicit (no braces) const add = (a, b) => a + b;
→ same result
Return object
Wrap in ( ) to avoid confusion
{ } is ambiguous — JS treats it as a function body. Wrap the object in ( ) to return it.
✗ wrong
const user = () => { name: "A" };
✓ correct
const user = () => ({ name: "A" });
No own this
Arrow borrows outer this
Regular functions get their own this. Arrow functions inherit this from the surrounding scope — important in React.
// regular — this = person const person = { name: "A", show: function() { console.log(this.name); } }; // arrow — this = outer scope const person = { name: "A", show: () => console.log(this) };
